
# Phoenix 导航页 - 设计思路

## 1. 整体风格与理念

Phoenix 导航页旨在提供一个**简约、高效且富有美感**的浏览器起始页。其核心设计理念是“宁静致远”，通过简洁的界面减少信息干扰，让用户能够专注于搜索和获取信息的核心任务。

- **简约主义**：移除了所有不必要的元素，只保留了核心的时间日期显示和搜索功能。
- **现代美学**：采用全屏背景、毛玻璃（半透明）效果、流畅的动画和清晰的字体，营造出现代感和科技感。
- **用户体验优先**：交互设计以直观、响应迅速为原则，通过动效反馈和快捷键提升操作效率。

## 2. 视觉设计 (UI)

### 2.1. 布局

页面采用**中心对称布局**，垂直方向分为两大功能区：

1.  **信息展示区（顶部）**：显示当前日期和实时更新的时间，作为页面的视觉锚点，帮助用户快速获取时间信息。
2.  **交互核心区（中部）**：包含搜索框和搜索引擎切换按钮，是用户操作的主要区域。

这种布局方式结构清晰，主次分明，符合用户自上而下的阅读习惯。

### 2.2. 色彩方案

- **主色调**：`#4285f4` (谷歌蓝)，用于搜索按钮、选中状态和关键交互元素，代表着科技、高效和可靠。
- **辅助色**：
    - `白色 (#ffffff)`：用于日期、时间和未选中的按钮文字，在深色背景下保证了良好的可读性。
    - `深灰色 (#333333)`：用于搜索框输入文字，与半透明背景形成对比。
    - `半透明背景 (rgba(255, 255, 255, 0.7))`：用于搜索框，营造出“毛玻璃”质感，既能融入背景，又能清晰地分出交互区域。

### 2.3. 字体与排版

- **字体**：优先使用 `PingFang SC` (苹方) 和 `Microsoft YaHei` (微软雅黑)，确保在不同操作系统上都有清晰、优雅的中文显示效果。
- **字重**：时间使用粗体 (700)，日期使用常规体 (300)，形成视觉对比，突出时间的重要性。
- **间距**：元素之间保持了足够的间距，避免了视觉拥挤，营造出轻松、开阔的视觉感受。

### 2.4. 背景

- **动态背景**：使用一张高质量的风景图片 (`background.png`) 作为背景，并覆盖一层从上到下的黑色渐变，增强了顶部文字的可读性，也让整体视觉更具深度。

## 3. 交互设计 (UX) & 动画

动画是提升用户体验的关键。本项目中的动画遵循**快速、平滑且有意义**的原则。

- **加载动画**：
    - **浮入效果 (`floatUp`)**：页面核心元素（信息区、搜索区）从下往上浮入并渐显，过程平滑自然，避免了页面元素的突兀出现。
    - **淡入效果 (`fadeIn`)**：各元素依次淡入，创建了富有层次感的加载序列。

- **交互动画**：
    - **搜索框聚焦 (`focused`)**：当用户点击搜索框时，边框会产生“呼吸灯”式的光晕效果，给予清晰的视觉反馈。
    - **按钮点击效果**：
        - **搜索引擎按钮**：点击时产生**波纹 (`ripple`)** 效果，反馈迅速且优雅。
        - **搜索按钮**：点击时有**缩放 (`scale`)** 效果，模拟真实的物理按压感。
    - **悬停效果 (`hover`)**：按钮在鼠标悬停时会轻微上浮或放大，并改变颜色，引导用户进行交互。

- **性能优化**：
    - **`will-change` 属性**：提前告知浏览器哪些元素的属性即将发生变化，优化动画性能，使其更流畅。
    - **`debounce` 函数**：用于监听窗口大小变化事件，防止高频触发，提升性能。

## 4. 功能实现

### 4.1. 核心功能

- **动态时间日期**：通过 JavaScript `Date` 对象获取并实时更新，每秒刷新一次。
- **多引擎搜索**：
    - 支持 Google, 百度, 必应, 神马四个主流搜索引擎。
    - 用户选择的搜索引擎会被记录在 `localStorage` 中，实现**记忆功能**，下次打开时自动设为默认。
- **响应式设计**：通过 `@media` 查询，适配不同尺寸的设备（PC、平板、手机），在小屏幕上会自动调整字体大小、元素间距和布局。

### 4.2. 快捷键

为了提升效率，设计了以下快捷键：

- **`Alt + 1/2/3/4`**：快速切换到对应的搜索引擎。
- **`Esc`**：清空搜索框内容并重新聚焦，方便用户进行下一次搜索。

### 4.3. 兼容性与优化

- **浏览器兼容性**：代码使用了现代 CSS 和 JavaScript 特性，主要兼容主流现代浏览器。
- **移动端视口 (`vh`) 优化**：通过 JavaScript 计算并设置 `--vh` CSS 变量，解决了移动端浏览器 `100vh` 高度计算不准确的问题（通常由地址栏的显示和隐藏引起）。
